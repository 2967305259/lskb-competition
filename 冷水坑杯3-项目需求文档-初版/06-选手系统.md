# 选手系统

## 选手 Dashboard

路由：`GET /player/dashboard`

### 页面内容
1. **待处理邀请**：来自队长的队伍邀请，可接受/拒绝
2. **当前队伍信息**：如果已加入队伍，显示队伍详情和成员
3. **比赛状态概览**：如果已匹配，显示当前比赛信息
4. **操作引导**：根据当前 status 引导下一步操作

## 邀请管理

### 接收邀请
路由：`GET /player/invitations`
- 分页展示收到的所有邀请
- 显示邀请队伍、发送时间、状态
- 支持接受（POST `/player/invitation/<id>/accept`）和拒绝（POST `/player/invitation/<id>/reject`）

### 接受邀请流程
```
接受邀请 → invitation.status = 'accepted'
→ 创建 TeamMember 记录
→ user.status = 1（已入队）
→ 发送通知给队长
```

### 拒绝邀请流程
```
拒绝邀请 → invitation.status = 'rejected'
→ 保持 user.status = 0
```

## 队伍信息查看

路由：`GET /player/team/<team_id>`
- 查看所在队伍的详细信息
- 队伍名、口号、照片、成员列表
- 每位成员的肉鸽选择、宣言

## 肉鸽主题选择

路由：`GET/POST /player/team/roguelike`
- 从可用肉鸽列表中选择一个主题
- 受 `max_per_team` 限制（如每队每种肉鸽最多1人）
- 选择后：`TeamMember.roguelike = <key>`，`user.status = 2`（已配置）

## 参赛宣言

路由：`GET/POST /player/declaration`
- 填写/修改个人参赛宣言（最多200字符）
- 存储在 `TeamMember.declaration`

## 时间偏好设置

路由：`GET/POST /player/time-preferences`
- 设置每周可用的时间段
- 格式：星期几 + 开始时间 + 结束时间
- 设置后：`user.status = 3`（已设置时间）
- **⚠️ 当前问题**：已收集但自动匹配算法未使用

## 选手个人主页

路由：`GET /player/profile`
- 头像显示（支持上传修改）
- 个人信息：用户名、昵称、角色、注册时间
- 个人积分：`TeamMember.final_score`
- 最近比赛记录（最近10场）
- 比赛统计：胜/负/平/总场次

## 选手 API

### GET `/player/api/players`
获取所有选手列表（JSON），含队伍信息

### GET `/player/api/players/<int:user_id>`
获取单个选手详情（JSON），含比赛统计

### GET `/player/api/players/<name>`
按用户名或昵称搜索选手（JSON）

## 选手系统不足与改进建议

1. **缺少选手公开主页**：目前选手个人主页仅自己可见，应有公开的个人资料页
2. **比赛历史不完整**：profile 页面仅显示最近10场，缺少完整的个人比赛历史
3. **肉鸽选择不可修改**：选择肉鸽后无修改机制（需管理员介入）
4. **无选手退出队伍机制**：选手无法主动退出队伍
5. **JoinRequest 未实现**：选手无法主动申请加入队伍（只能被动等待邀请）
