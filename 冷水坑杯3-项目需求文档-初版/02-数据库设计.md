# 数据库设计

## 数据库选择
- **开发环境**：SQLite（`。db/lskb.db`）
- **生产环境**：MySQL 8（通过 `DATABASE_URL` 环境变量切换）
- **迁移工具**：Flask-Migrate / Alembic

## 数据模型

### 1. Tournament（赛事表 `tournaments`）
| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | Integer PK | 主键 |
| `name` | String(100) | 赛事名称 |
| `description` | Text | 赛事描述 |
| `season_number` | Integer | 赛季编号（默认1） |
| `is_active` | Boolean | 是否为当前活跃赛事 |
| `status` | String(20) | 赛事状态：draft / registration / matching / running / finished |
| `registration_start` | DateTime | 报名开始时间 |
| `registration_end` | DateTime | 报名截止时间 |
| `tournament_start` | DateTime | 比赛开始时间 |
| `tournament_end` | DateTime | 比赛结束时间 |
| `bonus_score` | Integer | 队伍总分系数（默认500） |
| `created_at` | DateTime | 创建时间 |

**关联**：teams, matches, announcements, carousel_items

---

### 2. User（用户表 `users`）
| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | Integer PK | 主键 |
| `username` | String(20) | 用户名，唯一，索引 |
| `nickname` | String(20) | 游戏昵称，唯一 |
| `password_hash` | String(256) | Werkzeug 密码哈希 |
| `role` | String(20) | 角色：super_admin / admin / captain / player |
| `approval_status` | String(20) | 审批状态：pending / approved / rejected |
| `password_changed` | Boolean | 是否已修改默认密码（强制修改机制） |
| `avatar` | String(500) | 头像文件名 |
| `status` | Integer | 选手状态机：0=等待邀请 1=已入队 2=已配置 3=已设置时间 4=已匹配 5=比赛中 6=已提交结果 |
| `has_completed_onboarding` | Boolean | 是否已完成新手引导 |
| `created_at` | DateTime | 注册时间 |
| `updated_at` | DateTime | 更新时间 |

**关联**：team（一对一，作为队长），team_memberships（一对多），invitations（一对多）

### 7. Setting（系统设置表 `settings`）- 单例模式
| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | Integer PK | 主键 |
| `registration_open` | Boolean | 报名是否开放（默认True） |
| `tournament_status` | String(20) | 赛事状态 |

### 8. Roguelike（肉鸽主题表 `roguelikes`）
| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | Integer PK | 主键 |
| `key` | String(30) | 唯一标识（如 water、sami） |
| `name` | String(50) | 显示名称 |
| `is_active` | Boolean | 是否启用 |
| `max_per_team` | Integer | 每队最多选择人数（默认1） |
| `sort_order` | Integer | 排序权重 |
| `difficulty_min` | Integer | 最低难度（默认12） |
| `difficulty_max` | Integer | 最高难度（默认18） |
| `difficulty_step` | Float | 每级难度系数增量（默认0.1） |
| `created_at` | DateTime | 创建时间 |

**默认数据**：
- `water` — 水月与深蓝之树
- `sami` — 探索者的银凇止境
- `sarkaz` — 萨卡兹的无终奇语
- `garden` — 岁的界园志异

**难度系数公式**：`multiplier = 1.0 + (所选难度 - difficulty_min) * difficulty_step`

### 9. Match（比赛表 `matches`）
| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | Integer PK | 主键 |
| `tournament_id` | Integer FK → tournaments.id | 所属赛事 |
| `player_a_id` | Integer FK → users.id | 选手A |
| `player_b_id` | Integer FK → users.id | 选手B |
| `team_a_id` | Integer FK → teams.id | 选手A所属队伍 |
| `team_b_id` | Integer FK → teams.id | 选手B所属队伍 |
| `roguelike` | String(20) | 肉鸽主题 |
| `difficulty` | Integer | 统一难度（遗留字段，默认12） |
| `player_a_difficulty` | Integer | 选手A独立难度 |
| `player_b_difficulty` | Integer | 选手B独立难度 |
| `status` | String(20) | 比赛状态 |
| `player_a_result` | String(20) | A的结果：win / lose / draw / forfeit |
| `player_b_result` | String(20) | B的结果 |
| `player_a_score` | Integer | A提交的原始分数 |
| `player_b_score` | Integer | B提交的原始分数 |
| `winner_id` | Integer FK → users.id | 胜者ID |
| `confirmed_a` | Boolean | A是否确认约定时间 |
| `confirmed_b` | Boolean | B是否确认约定时间 |
| `review_status` | String(20) | 管理员审核状态：pending / approved / rejected |
| `review_reason` | Text | 审核理由 |
| `scheduled_time` | DateTime | 约定比赛时间 |
| `match_note` | Text | 选手间聊天/备注 |
| `created_at` | DateTime | 创建时间 |
| `finished_at` | DateTime | 完成时间 |

**比赛状态流转**：
```
pending → scheduled → confirmed → submitted → finished
  ↓         ↓           ↓           ↓
forfeit   forfeit     forfeit     dispute
```

## ER 关系图（文字描述）

```
Tournament (1) ──< (N) Team
Tournament (1) ──< (N) Match
Tournament (1) ──< (N) Announcement
Tournament (1) ──< (N) CarouselItem

User (1) ──< (N) TeamMember
User (1) ──< (N) Invitation
User (1) ──< (N) Notification
User (1) ──< (N) TimePreference
User (1) ── (1) Team (as captain)

Team (1) ──< (N) TeamMember
Team (1) ──< (N) Invitation
Team (1) ──< (N) Match (as team_a / team_b)

Match (1) ──< (N) ScoreLog

Setting (1) — 单例表，全局唯一

Roguelike — 独立表，被 TeamMember 和 Match 引用
```

## 数据库改进建议
1. **删除 Setting 表冗余**：将 tournament_status、tournament_name 等整合到 Tournament 模型
2. **为 TeamMember 添加 position 字段**：支持原设计的冷/正常1/正常2/坑位 4位置体系
3. **激活未连线模型**：JoinRequest、Announcement、CarouselItem、SystemLog 需实现路由和 UI
4. **添加索引**：matches.status、matches.created_at、teams.team_score 建议添加索引
5. **通知表增强**：添加 notification_type 字段便于分类展示
6. **统一难度字段**：移除 Match.difficulty 遗留字段，统一使用 player_a/b_difficulty
