# 比赛系统

## 赛事生命周期

```
registration (报名) → matching (匹配中) → running (比赛中) → finished (已结束)
```

### 状态转换
| 转换 | 触发操作 | 路由 |
|---|---|---|
| registration → matching | 管理员启动赛事 | `POST /admin/start-tournament` |
| matching → running | 管理员执行自动匹配 | `POST /admin/run-matching` |
| running → finished | 管理员结束赛事 | `POST /admin/finish-tournament` |
| 任意 → registration | 管理员重置 | `POST /admin/resume-registration` |

### 启动赛事验证
执行 `start_tournament` 时验证所有队伍：
- 每队必须有4名成员
- 每位成员必须已选择肉鸽主题
- 验证失败则不允许进入匹配阶段

## 自动匹配算法

### 当前实现（`_auto_match`）
1. 获取所有已入队且已选肉鸽的选手
2. 按肉鸽主题分组
3. 在每个分组内，随机配对跨队伍选手
4. 创建 Match 记录（status='pending'）
5. 发送通知给配对选手

### 匹配规则
- 同队选手不互相对战
- 同肉鸽主题选手互相匹配
- 如某肉鸽分组人数为奇数，最后一人本轮轮空

### ⚠️ 时间偏好未利用
选手设置了 `time_preferences`，但匹配算法完全未使用这一数据。理想情况下应优先配对时间偏好重叠度高的选手。

## 比赛生命周期

```
pending → scheduled → confirmed → submitted → finished
  ↓         ↓           ↓           ↓
forfeit   forfeit     forfeit     dispute
```

### 各状态说明
| 状态 | 含义 | 操作者 |
|---|---|---|
| `pending` | 自动匹配创建，等待约定时间 | 系统 |
| `scheduled` | 一方提出了比赛时间 | 选手 |
| `confirmed` | 双方确认了比赛时间 | 选手 |
| `submitted` | 至少一方提交了比赛结果 | 选手 |
| `finished` | 比赛结果已确认 | 系统/管理员 |
| `dispute` | 双方提交结果不一致 | 系统自动标记 |
| `forfeit_a` / `forfeit_b` | 某方弃权 | 选手 |
| `reviewing` | 管理员审核中 | 管理员 |

## 选手比赛操作

### 约定时间
路由：`GET/POST /tournament/match/<id>/schedule`
- 选手提议比赛时间
- 状态从 pending 变为 scheduled
- 发送通知给对手

### 确认时间
路由：`POST /tournament/match/<id>/confirm-schedule`
- 确认对手提议的时间
- 双方都确认后状态变为 confirmed

### 提交结果
路由：`GET/POST /tournament/match/<id>/submit-result`
- 提交：原始分数（整数）+ 所选难度（12-18）+ 胜负判定（win/lose/draw）
- 自动判断逻辑（`_check_match_result`）：
  - A提交win + B提交lose → 自动结束，A胜
  - 双方都提交draw → 自动结束，平局
  - 任何不一致 → 标记为 dispute

### 弃权
路由：`POST /tournament/match/<id>/forfeit`
- 自动判定对手获胜
- 更新双方积分

### 比赛备注/聊天
路由：`POST /tournament/match/<id>/note`
- 选手间可在比赛页留言（带时间戳）

### 我的比赛
路由：`GET /tournament/my-matches`
- 查看当前用户作为选手A或选手B的所有比赛

### 比赛详情
路由：`GET /tournament/match/<match_id>`
- 仅参赛选手和管理员可查看
- 显示比赛完整信息、状态、双方操作按钮

## 管理员比赛管理

### 比赛列表
路由：`GET /admin/matches`
- 所有比赛列表，支持按状态筛选
- 搜索功能

### 争议处理
路由：`POST /admin/dispute/<id>/resolve`
- 管理员裁决：判A胜 / 判B胜 / 判平局
- 更新积分
- 通知双方选手

### 结果编辑
路由：`POST /admin/match/<id>/edit-result`
- 修改比赛结果、分数、难度
- 记录到 ScoreLog（操作人、旧值、新值、原因）
- 触发全量积分重算

### 审核比赛
路由：`POST /admin/match/<id>/review`
- 对已完成的比赛进行审核（approved/rejected）
- 填写审核理由

### 积分修改日志
路由：`GET /admin/match/<id>/score-logs`
- 查看该比赛的所有积分修改记录

## 积分计算

### 单场比赛积分
```
player_final_score = raw_score × difficulty_multiplier
difficulty_multiplier = 1.0 + (所选难度 - roguelike.difficulty_min) × roguelike.difficulty_step
```

### 队伍总分
```
team_score = Σ(所有成员 final_score) × tournament.bonus_score
```

## 比赛系统不足与改进建议

1. **匹配算法未使用时间偏好**：应整合 time_preferences 数据提升匹配质量
2. **无轮空补偿**：奇数人数分组时轮空选手无积分补偿
3. **匹配透明度不足**：选手无法看到自己被匹配的原因
4. **无比赛录像/截图上传**：结果提交缺乏证据附件功能
5. **比赛重赛机制缺失**：争议处理后无重新比赛机制
6. **比赛时限**：无比赛自动超时/强制结束机制
7. **批量操作**：管理员缺少批量审核/批量解决争议的功能
