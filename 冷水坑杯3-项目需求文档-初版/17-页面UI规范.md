# 页面 UI 规范

## 设计主题

整体采用 **明日方舟（Arknights）暗色主题**风格：
- 主色调：深色背景 + 金色/暖色点缀
- 卡片式布局，圆角设计
- 简洁有力的排版

## 技术栈

| 技术 | 版本 | 用途 |
|---|---|---|
| Bootstrap | 5.x | 响应式布局、组件库 |
| Font Awesome | - | 图标系统 |
| 自定义 CSS | - | 暗色主题覆盖、项目特定样式 |
| 自定义 JS | - | 轮播图、交互逻辑 |

## 布局规范

### 全局布局（base.html）
- 固定顶部导航栏（暗色）
- 主内容区：container / container-fluid
- 底部页脚（暗色）
- 响应式断点：Bootstrap 默认（sm/md/lg/xl）

### 导航栏
- 品牌区域：赛事名称 + Logo
- 主导航：首页 / 赛事大厅 / 队伍列表 / 规则
- 右侧：通知铃铛（未读角标）+ 用户下拉菜单
- 移动端：折叠汉堡菜单

### 卡片组件
- 圆角卡片（border-radius: 0.5rem）
- 暗色背景 + 微亮边框
- 内边距：1rem - 1.5rem
- 卡片间间距：1rem (g-3/g-4)

### 表格
- Bootstrap table 样式 + 暗色主题覆盖
- 斑马纹（table-striped）
- 悬停高亮（table-hover）
- 响应式水平滚动（table-responsive）

### 表单
- Bootstrap form 控件 + 暗色主题
- 标签在上，输入框在下
- 校验反馈：绿色（成功）/ 红色（失败）
- CSRF 保护（Flask-WTF 自动注入）

### 按钮
- Bootstrap btn 组件
- 主要操作：btn-primary（金色系）
- 危险操作：btn-danger（红色）
- 次要操作：btn-secondary / btn-outline-*

### 状态标签
- 赛事状态 / 比赛状态 / 审批状态 使用彩色 Badge
- pending/registration：蓝色
- approved/running/finished：绿色
- rejected/dispute/forfeit：红色
- scheduled：黄色

## UI 规范不足与改进建议

1. **配色标准化**：建议提取 CSS 变量，统一暗色主题色板
2. **组件库建设**：抽取可复用组件（状态标签、空状态、加载中、确认弹窗）
3. **移动端体验**：部分管理后台页面移动端体验需优化
4. **可访问性**：缺少 ARIA 标签、键盘导航支持
5. **加载状态**：缺少统一的 loading/骨架屏状态
6. **空状态设计**：列表为空时缺少友好的空状态提示
7. **Toast 通知**：缺少操作反馈 Toast（成功/失败提示）
8. **面包屑导航**：深层页面缺少面包屑返回路径
