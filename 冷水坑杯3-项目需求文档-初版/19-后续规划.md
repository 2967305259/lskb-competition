# 后续规划（RoadMap）

## 当前版本状态

**版本**：冷水坑杯 #3（v3.0）
**状态**：核心功能已实现，部分功能待完善

---

## Phase 1：基础功能完善（短期 - 1-2周）

### 🔴 高优先级（阻塞性问题）
| 任务 | 说明 |
|---|---|
| **通知列表页** | 实现 `/notifications` 路由，支持查看、标记已读、全部已读 |
| **JoinRequest 连线** | 实现选手主动申请加入队伍功能 |
| **队员移除功能** | 队长可移除队员 |
| **选手退出队伍** | 选手可主动退出队伍 |
| **肉鸽选择修改** | 允许选手在截止前修改肉鸽选择 |
| **匹配算法优化** | 将时间偏好数据纳入自动匹配逻辑 |

### 🟡 中优先级（体验提升）
| 任务 | 说明 |
|---|---|
| **公告系统连线** | Announcement CRUD + 首页/Tournament Hall 展示 |
| **轮播图管理** | CarouselItem 后台管理页面 |
| **系统日志连线** | SystemLog 写入 + 管理后台查看 |
| **操作确认弹窗** | 危险操作增加二次确认 |
| **Toast 操作反馈** | 统一的成功/失败提示 |

---

## Phase 2：架构优化（中期 - 3-4周）

### 代码重构
| 任务 | 说明 |
|---|---|
| **模块化拆分** | 将 app.py 拆分为 models/routes/forms/services 模块 |
| **API 标准化** | 统一 `/api/v1/` 前缀、响应格式、错误码 |
| **API 文档** | OpenAPI/Swagger 文档生成 |
| **配置独立** | 抽取 config.py 独立配置模块 |

### 功能扩展
| 任务 | 说明 |
|---|---|
| **个人排行榜** | 个人积分排名（跨队伍） |
| **肉鸽主题排行榜** | 各肉鸽主题独立排名 |
| **队伍内部留言板** | 队伍成员间沟通 |
| **比赛证据上传** | 截图上传功能 |
| **邮件通知** | 关键事件邮件提醒 |
| **密码找回** | 邮箱验证 + 密码重置 |

### 数据增强
| 任务 | 说明 |
|---|---|
| **位置系统** | TeamMember 添加 position 字段，实现冷/正常1/正常2/坑位 |
| **special_score 实现** | 特殊积分维度（如全胜奖励） |
| **minor_score 实现** | 辅助积分维度（如净胜分） |
| **赛季历史** | 保留往届赛事数据并可查看 |
| **排行榜导出** | 排行榜 CSV/Excel 导出 |

## Phase 4：技术债务清理（持续）

| 任务 | 说明 |
|---|---|
| **添加单元测试** | pytest + pytest-flask，覆盖率 ≥ 70% |
| **添加集成测试** | 关键用户流程的端到端测试 |
| **代码质量检查** | flake8/black/mypy 配置 + pre-commit hooks |
| **CI/CD 流程** | GitHub Actions 自动化测试和部署 |
| **性能优化** | 数据库查询优化、缓存引入 |
| **日志系统** | 结构化日志 + 日志级别管理 |
| **依赖管理** | 迁移到 pyproject.toml / Poetry |
| **Docker 化** | Dockerfile + docker-compose 部署方案 |
| **安全审计** | 依赖漏洞扫描、SQL 注入检查、XSS 防护 |

## 优先级矩阵

```
                    紧急度
                高        低
           ┌──────────┬──────────┐
      高   │ 通知列表   │ API标准化  │
           │ JoinReq   │ 模块拆分   │
重         │ 匹配优化   │ 个人排名   │
要   ├──────────┼──────────┤
性      │ 公告连线   │ 测试代码   │
      低   │ 轮播管理   │ CI/CD     │
           │ 队员移除   │ 移动端PWA │
           └──────────┴──────────┘
```

**建议优先完成左上角象限的任务（高重要性 + 高紧急度），它们直接影响当前赛事的正常运营。**
