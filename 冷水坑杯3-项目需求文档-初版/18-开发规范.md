## 命名规范

### Python
- **文件名**：snake_case（如 `match_service.py`）
- **类名**：PascalCase（如 `MatchForm`、`ScoreService`）
- **函数/方法**：snake_case（如 `get_active_tournament()`）
- **变量**：snake_case（如 `team_score`）
- **常量**：UPPER_SNAKE_CASE（如 `MAX_TEAM_SIZE`）

### 数据库
- **表名**：小写复数（如 `users`、`teams`、`matches`）
- **字段名**：snake_case（如 `team_score`、`created_at`）
- **主键**：`id`
- **外键**：`<related_table>_id`（如 `captain_id`、`tournament_id`）

### 模板
- **模板文件**：snake_case（如 `match_detail.html`）
- **块名**：snake_case（如 `{% block page_title %}`）
- **变量**：snake_case（如 `{{ unread_count }}`）

### 路由
- **URL**：kebab-case（如 `/submit-result`、`/my-matches`）
- **端点名**：snake_case（如 `match_detail`、`schedule_match`）

## 数据库规范

- 使用 SQLAlchemy ORM，避免原始 SQL
- 所有外键关系使用 `db.relationship` 声明
- 模型提供 `to_dict()` 方法便于 JSON 序列化
- 唯一约束使用命名约束（如 `uq_team_user`）
- 迁移文件由 Flask-Migrate 自动生成，提交到版本管理

## 表单规范

- 所有表单继承 `FlaskForm`
- 使用 DataRequired、Length、Email 等验证器
- 自定义验证器以 `validate_` 前缀命名
- CSRF 保护默认启用

## 认证与安全

- 密码使用 `werkzeug.security.generate_password_hash` 加密
- 使用 `@login_required` 装饰器保护需要认证的路由
- 角色检查在 `before_request` 或路由内实现
- 敏感操作使用 POST 方法（GET 仅用于查询）
- Flask-WTF CSRF 保护所有 POST 请求

## 错误处理

- 自定义 403、404、500 错误页面
- API 端点返回 JSON 格式错误信息
- 用户可见的错误信息使用中文
- 敏感错误信息不暴露给用户（仅日志）

## Git 规范

### 分支策略
- `main`：稳定版本（当前生产环境）
- `develop`：开发分支
- `feature/<name>`：功能分支
- `fix/<name>`：修复分支

### 提交信息
```
<type>: <简短描述>

<详细说明（可选）>
```

Type 类型：
- `feat`: 新功能
- `fix`: 修复 bug
- `refactor`: 代码重构
- `style`: 样式修改
- `docs`: 文档更新
- `test`: 测试相关
- `chore`: 构建/工具链

## 当前不足

1. **无测试代码**：项目完全没有单元测试或集成测试
2. **单体文件**：app.py 超过 3000 行需拆分
3. **无代码检查**：缺少 linting（flake8/pylint）和格式化（black）配置
4. **无类型注解**：Python 代码缺少类型提示
5. **无 CI/CD**：缺少自动化构建/测试/部署流程
6. **无环境管理**：缺少 pyproject.toml 或 Pipfile 等依赖管理文件
7. **docstring 不全**：大部分函数缺少文档字符串
8. **错误日志**：缺少集中式日志记录（仅 flask 默认日志）
