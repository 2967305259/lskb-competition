# 后台管理

## 管理入口

路由：`GET /admin/dashboard`

### Dashboard 概览
- 用户总数、队伍总数、比赛总数
- 当前赛事状态
- 待审批用户数
- 各状态比赛分布
- 快速操作入口

## 用户管理

### 用户列表
路由：`GET /admin/users`
- 分页展示所有用户
- 支持搜索（用户名/昵称）
- 显示：用户名、昵称、角色、审批状态、注册时间

### 角色管理
| 操作 | 路由 | 权限要求 |
|---|---|---|
| 升级为队长 | `POST /admin/user/<id>/promote` | admin |
| 降级为选手 | `POST /admin/user/<id>/demote` | admin |
| 升级为管理员 | `POST /admin/user/<id>/promote-admin` | super_admin |
| 降级管理员 | `POST /admin/user/<id>/demote-admin` | super_admin |

### 删除用户
路由：`POST /admin/user/<id>/delete`
- 如果是队长，同时删除其队伍
- 删除其 TeamMember、Invitation 等关联数据

### 用户审批
路由：`GET /admin/approvals`
- 筛选：待审批 / 已通过 / 已拒绝
- 操作：`POST /admin/user/<id>/approve` 和 `/reject`
- 审批后发送通知给用户

## 队伍管理

### 队伍列表
路由：`GET /admin/teams`
- 所有队伍列表
- 支持搜索

### 队伍详情
路由：`GET /admin/team/<team_id>`
- 队伍完整信息
- 成员列表及个人积分

### 积分修改
路由：`POST /admin/team/<id>/update-scores`
- 手动编辑每位成员的 final_score
- 自动重算队伍总分

### 删除队伍
路由：`POST /admin/team/<id>/delete`
- 删除队伍及相关数据

## 报名管理

### 报名控制
路由：`GET/POST /admin/registration`
- 开关：设置 `registration_open` 为 True/False
- 控制新用户注册

### 报名列表
路由：`GET /admin/registration-list`
- 所有已报名队伍列表

### Excel 导出
路由：`GET /admin/export`
- 导出格式：.xlsx
- 包含：队伍信息、成员列表、肉鸽选择、宣言

## 赛事流程控制

### 启动赛事
路由：`POST /admin/start-tournament`
- registration → matching
- 验证所有队伍满员且已选择肉鸽

### 执行匹配
路由：`POST /admin/run-matching`
- matching → running
- 执行自动匹配算法

### 结束赛事
路由：`POST /admin/finish-tournament`
- running → finished
- 生成最终排名

### 重置赛事
路由：`POST /admin/resume-registration`
- 任意状态 → registration
- 清空匹配数据

## 比赛管理

### 比赛列表
路由：`GET /admin/matches`
- 所有比赛，支持按状态筛选

### 争议处理
路由：`GET /admin/disputes`
- 所有争议中比赛列表
- `POST /admin/dispute/<id>/resolve`：裁决

### 比赛审核
路由：`POST /admin/match/<id>/review`
- 审核已完成比赛（approved/rejected）

### 结果编辑
路由：`POST /admin/match/<id>/edit-result`
- 修改结果、分数、难度
- 记录 ScoreLog

### 查看修改日志
路由：`GET /admin/match/<id>/score-logs`

## 肉鸽管理

### 肉鸽列表
路由：`GET /admin/roguelikes`

### 新增/编辑/删除
路由：`POST /admin/roguelike/add` 等

## 积分配置

路由：`GET/POST /admin/score-config`
- 胜负平积分
- 难度系数（已迁移到肉鸽模型）
- 晋级数量
- 重置积分

## 晋级管理

路由：`GET /admin/advance`
- 设置晋级数量
- 查看晋级队伍

## 后台管理不足与改进建议

1. **缺少操作审计**：SystemLog 模型已有但未使用，管理员关键操作无日志
2. **批量操作缺失**：无批量审批、批量删除等功能
3. **数据导出单一**：仅 Excel 方式，缺少 CSV 等格式
4. **仪表盘数据简单**：可增加图表可视化（比赛进度、积分分布等）
5. **操作确认不足**：部分危险操作（如删除队伍、结束赛事）缺少二次确认
6. **缺少公告管理**：Announcement 模型已有但无 CRUD 页面
7. **缺少轮播图管理**：CarouselItem 模型已有但无管理界面
8. **缺少系统日志查看**：SystemLog 模型已有但无查看页面
