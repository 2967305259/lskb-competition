# 肉鸽系统

## 概述

肉鸽（Roguelike）系统管理《明日方舟》集成战略的不同主题模式。选手选择肉鸽主题后，同肉鸽主题的选手会被匹配对战。

## 默认肉鸽主题

| Key | 名称 | 最低难度 | 最高难度 | 难度步长 | 每队上限 |
|---|---|---|---|---|---|
| `water` | 水月与深蓝之树 | 12 | 18 | 0.1 | 1 |
| `sami` | 探索者的银凇止境 | 12 | 18 | 0.1 | 1 |
| `sarkaz` | 萨卡兹的无终奇语 | 12 | 18 | 0.1 | 1 |
| `garden` | 岁的界园志异 | 12 | 18 | 0.1 | 1 |

## 肉鸽数据模型

```python
class Roguelike(db.Model):
    key              # 唯一标识
    name             # 显示名称
    is_active        # 是否启用
    max_per_team     # 每队最多选择人数
    sort_order       # 排序权重
    difficulty_min   # 最低难度
    difficulty_max   # 最高难度
    difficulty_step  # 难度系数增量
```

## 难度系数公式

```python
def get_multiplier(difficulty):
    return 1.0 + (difficulty - self.difficulty_min) * self.difficulty_step
```

## 选手选择肉鸽

路由：`GET/POST /player/team/roguelike`

### 选择规则
- 每队每种肉鸽最多 `max_per_team` 人选择（默认1人）
- 选手只能选择一种肉鸽主题
- 选择后更新 `TeamMember.roguelike` 和 `user.status = 2`
- 队长同样需要选择肉鸽

### 限制校验
- 提交时检查该肉鸽是否已被同队其他成员选满
- 如果超限，返回错误提示

## 管理后台 CRUD

路由：`GET /admin/roguelikes`

### 功能
- **列表**：查看所有肉鸽主题（含启用/禁用状态）
- **新增**：`POST /admin/roguelike/add`
  - 设置 key、name、max_per_team、难度范围、步长
- **编辑**：`POST /admin/roguelike/<id>/edit`
  - 修改任意配置项
- **删除**：`POST /admin/roguelike/<id>/delete`
  - ⚠️ 需确认无选手已选择该主题

### 配置参数说明
- `key`：唯一标识符（英文，如 water、sami）
- `name`：中文显示名称
- `is_active`：禁用后选手不可选择
- `max_per_team`：每队限制人数
- `sort_order`：显示排序
- `difficulty_min`：可选最低难度
- `difficulty_max`：可选最高难度
- `difficulty_step`：每级难度系数增幅

## 肉鸽选择与匹配

自动匹配时按肉鸽主题分组：
1. 获取所有已选肉鸽的选手
2. 按 `roguelike` 字段分组
3. 同组内随机配对（跨队伍）
4. 如某组人数为奇数，最后一人轮空

## 肉鸽系统不足与改进建议

1. **肉鸽选择不可修改**：选手选择后无法更改（需要管理员通过数据库修改）
2. **无肉鸽选择统计**：缺少各肉鸽主题选择人数的统计面板
3. **删除肉鸽无保护**：删除有选手已选择的肉鸽时缺乏明确警告
4. **肉鸽描述信息缺失**：模型缺少肉鸽玩法/规则描述字段
5. **每队配置不够灵活**：建议增加每队总肉鸽种类数限制（如：4人队伍最少选几种肉鸽）
6. **肉鸽历史版本**：不同赛季肉鸽数据无版本隔离
