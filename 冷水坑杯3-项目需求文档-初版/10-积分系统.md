# 积分系统

## 积分计算模型

### 比赛基础积分
| 结果 | 默认积分 |
|---|---|
| 胜利 (win) | 2 分 |
| 平局 (draw) | 1 分 |
| 失败 (lose) | 0 分 |

基础积分可通过管理后台修改（`Setting.win_score` / `draw_score` / `lose_score`）。

### 难度系数加权
每位选手在提交比赛结果时选择自己的难度（12-18），系统根据所选肉鸽主题的难度配置计算系数：

```
难度系数 = 1.0 + (选手选择难度 - 肉鸽最低难度) × 难度步长
```

默认配置（以水月主题为例）：
- 难度12 → 系数 1.0
- 难度13 → 系数 1.1
- 难度14 → 系数 1.2
- 难度15 → 系数 1.3
- ...
- 难度18 → 系数 1.6

### 单场最终得分
```
选手单场得分 = 原始分数 × 难度系数
```

### 个人总积分
```
选手 final_score = Σ 所有已完成比赛的单场得分
```

### 队伍总积分
```
队伍 team_score = Σ 所有队员 final_score × 赛事 bonus_score 系数（默认500）
```

## 排名规则

排名按以下优先级排序（降序）：
1. `team_score`（队伍总分）
2. `special_score`（特殊积分，当前未使用，始终为0）
3. `minor_score`（辅助积分，当前未使用，始终为0）

## 积分管理

### 管理后台积分配置
路由：`GET/POST /admin/score-config`

可配置项：
- 胜负平基础积分
- 各难度系数（历史遗留，现已迁移到 Roguelike 模型）
- 晋级队伍数量
- 赛事奖励系数

### 手动调整积分
路由：`POST /admin/team/<id>/update-scores`
- 管理员可直接修改每位队员的 `final_score`
- 提交后自动重新计算队伍总分

### 积分重置与重算
路由：`POST /admin/reset-scores`
- 将所有选手积分归零
- 遍历所有已完成比赛，按公式重新计算
- 重新计算所有队伍总分

### 积分修改审计
- `ScoreLog` 表记录每次管理员手动修改
- 记录修改人、旧值、新值、原因、时间
- 可在比赛详情页查看修改历史

## 排行榜展示

### 首页 TOP 10
- 显示前10名队伍
- 排名、队伍名、积分

### 赛事大厅
路由：`GET /tournament/`
- 完整比赛统计数据
- 实时排行榜
- 晋级队伍展示
- 最近比赛结果

### 完整排行榜
路由：`GET /tournament/rankings`
- 所有队伍排名
- 积分详情

### 最终结果
路由：`GET /tournament/results`
- 冠军、亚军、季军展示
- 赛事结束后可见

## 晋级管理

路由：`GET /admin/advance`
- 管理员设置晋级队伍数量（`advance_count`）
- 自动根据排名截取前 N 名
- 在赛事大厅展示晋级队伍

## 积分系统不足与改进建议

1. **special_score 和 minor_score 未使用**：排名有这两个维度但始终为0，应实现或移除
2. **无个人排名**：仅有队伍排名，无个人积分排行榜
3. **积分趋势图**：缺少队伍/个人积分变化趋势可视化
4. **难度验证缺失**：选手自行上报难度，无验证机制
5. **手动调整缺乏批量能力**：一次只能修改一个队伍
6. **原始分数含义不明确**：raw_score 是游戏内分数还是其他指标？建议在提交界面说明
7. **无赛季历史数据**：前两届赛事数据未保留/展示
8. **Excel 导出功能有限**：建议增加比赛结果导出、排名历史导出
