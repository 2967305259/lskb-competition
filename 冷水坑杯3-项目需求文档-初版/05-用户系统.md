# 用户系统

## 用户注册

### 注册条件
- 赛事报名开关 `registration_open` 必须为 True
- 用户名 2-20 字符，唯一
- 昵称 2-20 字符，唯一
- 密码由 Werkzeug `generate_password_hash` 加密存储

### 注册流程
```
访问 /auth/register → 填写表单 → 提交
→ approval_status = 'pending'
→ role = 'player'（默认，普通用户只能注册为选手）
→ 等待管理员审批
```

### 注册表单字段
- `username` — 用户名（String, 2-20字符）
- `nickname` — 游戏昵称（String, 2-20字符）
- `password` — 密码
- `confirm_password` — 确认密码

## 用户登录

### 登录流程
```
访问 /auth/login → 填写用户名+密码 → 提交
→ 检查 approval_status（未通过审批禁止登录）
→ 检查 password_changed（默认管理员首次登录强制修改密码）
→ 登录成功 → 按角色跳转对应首页
  - super_admin/admin → /admin/dashboard
  - captain → /captain/dashboard
  - player → /player/dashboard
```

### 登录路由表
| 角色 | 登录后跳转 |
|---|---|
| super_admin | `/admin/dashboard` |
| admin | `/admin/dashboard` |
| captain | `/captain/dashboard` |
| player | `/player/dashboard` |

## 选手状态机

选手的 `status` 字段（Integer 0-6）代表了从注册到完赛的完整生命周期：

```
0 (等待邀请)  → 接受邀请加入队伍
1 (已入队)    → 选择肉鸽主题 + 填写宣言
2 (已配置)    → 设置时间偏好
3 (已设置时间) → 管理员执行自动匹配
4 (已匹配)    → 比赛进行中
5 (比赛中)    → 提交比赛结果
6 (已提交结果)
```

## 用户审批

### 审批状态
- `pending` — 待审批（新注册默认状态）
- `approved` — 已通过，可正常登录使用
- `rejected` — 已拒绝，无法登录

### 审批操作
- 管理员在 `/admin/approvals` 页面查看待审批用户
- 支持按状态筛选：待审批 / 已通过 / 已拒绝
- 审批通过/拒绝后发送通知给用户

## 修改密码

- 路由：`GET/POST /auth/change-password`
- 需提供旧密码验证
- 修改后 `password_changed = True`（满足强制改密要求）

## 头像管理

- 上传路由：`POST /player/upload-avatar` 和 `POST /captain/upload-avatar`
- 支持的格式：PNG、JPG、GIF、WEBP
- 文件以 UUID 命名存储在 `app/static/uploads/avatars/`
- user.avatar 存储文件名

## 用户系统不足与改进建议

1. **无密码复杂度校验**：建议增加最少8位、包含数字+字母的要求
2. **无邮箱验证**：无法找回密码，建议增加邮箱字段和密码重置功能
3. **审批流程简陋**：无审批理由反馈给被拒绝用户
4. **无账号注销**：用户无法自行注销账号
5. **昵称修改功能缺失**：用户无法自行修改昵称
