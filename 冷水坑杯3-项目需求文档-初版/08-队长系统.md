# 队长系统

## 队长角色

队长是选手和队伍之间的桥梁角色，具有以下专属能力：
- 创建并管理一支队伍
- 搜索并邀请选手加入队伍
- 管理队伍信息和照片
- 作为 TeamMember 参与比赛

## 队长 Dashboard

路由：`GET /captain/dashboard`

### 页面内容
1. **队伍概览**：队伍名、口号、照片、成员数、创建时间
2. **待处理邀请**：已发送但未被处理的邀请列表
3. **队伍完整性状态**：成员是否已满4人、每位成员是否已完成配置
4. **操作引导**：提示下一步操作（如：队伍未满员则引导邀请队员）

## 队长特有操作

### 创建队伍
- 每位队长仅能创建一支队伍
- 创建后自动成为队伍成员
- 队伍名全局唯一

### 编辑队伍
- 可修改队伍名和口号
- 可上传队伍照片

### 邀请选手
- 搜索范围：所有已审批通过的选手
- 排除条件：已有队伍的选手、已被本队邀请的选手、队长本人
- 邀请后发送通知给选手

### 管理邀请
- 查看所有已发送的邀请
- 取消待处理的邀请（已接受/已拒绝的不可取消）

### 查看成员
- 查看队伍成员完整信息
- 每位成员的肉鸽选择和宣言
- 每位成员的个人积分

## 队长个人主页

路由：`GET /captain/profile`
- 同选手个人主页结构
- 额外显示：管理的队伍信息
- 头像上传

## 队长与选手的共享功能

队长本身也是选手（TeamMember），因此共享以下选手功能：
- 选择肉鸽主题（`/player/team/roguelike`）
- 填写参赛宣言（`/player/declaration`）
- 设置时间偏好（`/player/time-preferences`）
- 查看比赛和提交结果（`/tournament/*`）
- 上传头像

## 队长系统不足与改进建议

1. **无法移交队长**：队长无法将队长职位转让给队员
2. **无法移除队员**：队长无法移除非活跃/不合适的队员
3. **队伍解散功能缺失**：队长无法主动解散队伍
4. **无法拒绝选手主动申请**：JoinRequest 功能未实现，队伍被动接收申请不可行
5. **搜索选手功能简单**：建议增加按肉鸽偏好、时间匹配度等维度筛选
